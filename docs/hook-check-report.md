# OneNote Ribbon Formatting Rules Table

Date: 2026-04-17 (verified across 10 rounds of agent-team code review)
Repository: onenote-ribbon
Branch: main

## Scope

This table captures all formatting rules, editor behaviors, active-state detection, and user-visible interactions defined by the ribbon and its editor integration logic. Covers markdown/HTML transforms, list style injection, tag behaviors, clipboard actions, email export, style presets, and ribbon shell mechanics. All 110 non-test source files in `src/` were read.

## Rules Table

| ID  | Area                            | Rule                                      | Behavior                                                                                                                                                                                                                                                                                                                                                                    |
| --- | ------------------------------- | ----------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | Styling Engine — Domain         | Domain detection method                   | Checks whether any non-markdown HTML tag structurally **encloses** the cursor/selection (via upward DOM walk). A text-scan of selection content is NOT used; only enclosing tags trigger HTML domain.                                                                                                                                                                       |
| 2   | Styling Engine — Domain         | HTML domain selection                     | Enclosing tag name must appear in `htmlTagNames` list to set domain to `'html'`. `'code'` is classified as a markdown tag name and does NOT push to HTML domain.                                                                                                                                                                                                            |
| 3   | Styling Engine — Domain         | MD→HTML one-directional                   | `resolveTagForDomain` only substitutes markdown tags for HTML equivalents when in HTML domain. HTML tags are never converted back to markdown.                                                                                                                                                                                                                              |
| 4   | Styling Engine — Domain         | Domain conversion trigger                 | First HTML-format operation on markdown content converts the **outermost enclosing MD tag region** (not just selection bounds) from markdown delimiters to HTML equivalents before applying the new format.                                                                                                                                                                 |
| 5   | Styling Engine — Domain         | Domain conversion MD token set            | Converts: `***` → `<b><i>`, `**` → `<b>`, `__` → `<b>`, `*` → `<i>`, `_` → `<i>`, `~~` → `<s>`, `==` → `<mark>`. Entries are ordered longest-delimiter-first to prevent partial-match confusion.                                                                                                                                                                            |
| 6   | Styling Engine — Domain         | Domain conversion fallback                | If no enclosing MD tag ranges are found (e.g. only non-enclosing tokens present), conversion falls back to a plain HTML wrap of the selection.                                                                                                                                                                                                                              |
| 7   | Styling Engine — Domain         | Domain conversion cursor tracking         | After domain conversion, `StylingResult` emits `newSelectionStart`/`newSelectionEnd` so the editor can reposition selection. Active selection: finds `selectedText` substring in replacement string. Cursor only: placed after the opening markup.                                                                                                                          |
| 8   | Styling Engine — Tags           | Bold toggle (MD)                          | `text` → `**text**`; toggles off when already wrapped. `__text__` form is also recognized as bold (MD).                                                                                                                                                                                                                                                                     |
| 9   | Styling Engine — Tags           | Italic toggle (MD)                        | `text` → `*text*`; toggles off when already wrapped. `_text_` form is also recognized as italic (MD).                                                                                                                                                                                                                                                                       |
| 10  | Styling Engine — Tags           | Bold+Italic toggle (MD)                   | `***text***` is recognized as combined bold+italic; converted to `<b><i>` in HTML domain.                                                                                                                                                                                                                                                                                   |
| 11  | Styling Engine — Tags           | Strikethrough toggle (MD)                 | `text` → `~~text~~`; toggles off when already wrapped.                                                                                                                                                                                                                                                                                                                      |
| 12  | Styling Engine — Tags           | Highlight toggle (MD)                     | `text` → `==text==`; toggles off when already wrapped.                                                                                                                                                                                                                                                                                                                      |
| 13  | Styling Engine — Tags           | Underline toggle                          | Uses `<u>text</u>` (only HTML form; no markdown equivalent).                                                                                                                                                                                                                                                                                                                |
| 14  | Styling Engine — Tags           | Bold toggle (HTML)                        | Uses `<b>text</b>` — NOT `<strong>`.                                                                                                                                                                                                                                                                                                                                        |
| 15  | Styling Engine — Tags           | Italic toggle (HTML)                      | Uses `<i>text</i>` — NOT `<em>`.                                                                                                                                                                                                                                                                                                                                            |
| 16  | Styling Engine — Tags           | Strikethrough toggle (HTML)               | Uses `<s>text</s>` — NOT `<del>`.                                                                                                                                                                                                                                                                                                                                           |
| 17  | Styling Engine — Tags           | Highlight toggle (HTML)                   | Uses `<mark>text</mark>`.                                                                                                                                                                                                                                                                                                                                                   |
| 18  | Styling Engine — Tags           | Subscript toggle                          | Uses `<sub>text</sub>`.                                                                                                                                                                                                                                                                                                                                                     |
| 19  | Styling Engine — Tags           | Superscript toggle                        | Uses `<sup>text</sup>`.                                                                                                                                                                                                                                                                                                                                                     |
| 20  | Styling Engine — Tags           | Script exclusivity                        | Toggling subscript over an existing superscript (or vice versa) performs an **in-place markup swap** via `buildTagMarkupSwapReplacements` — it replaces the opening and closing tag text in one pass. No remove-then-wrap. Also checks delimiter-inclusive match for the swap.                                                                                              |
| 21  | Styling Engine — Tags           | Tag boundary cursor                       | A cursor placed exactly at `openingTagEndOffset` or `closingTagStartOffset` is treated as **inside** the tag (inclusive `<=`/`>=` comparison) and triggers removal.                                                                                                                                                                                                         |
| 22  | Styling Engine — Tags           | findEnclosingMatchingTag                  | Returns the **innermost** enclosing tag (smallest content width).                                                                                                                                                                                                                                                                                                           |
| 23  | Styling Engine — Tags           | findDelimiterInclusiveMatch               | Returns the **outermost** tag whose full span (including delimiter chars) fits within the selection.                                                                                                                                                                                                                                                                        |
| 24  | Styling Engine — Tags           | Delimiter-inclusive toggle/remove         | When the full tag span including delimiter chars falls within the selection, `unwrapTag` removes the delimiter chars themselves (not just content). Used in both `toggleTag` and `removeTag`.                                                                                                                                                                               |
| 25  | Styling Engine — Tags           | Span match property-name-only             | When matching span tags, only the CSS **property name** is compared (not value). A search for `color: blue` matches any enclosing span with a `color:` property of any value.                                                                                                                                                                                               |
| 26  | Styling Engine — Tags           | Same-CSS-property span replacement        | Applying a span format (color/size/font) when an enclosing span with the same CSS property already exists calls `replaceOpeningTagAttribute` to update the value in-place, preventing double-wrapping.                                                                                                                                                                      |
| 27  | Styling Engine — Tags           | removeTag delimiter-inclusive fallback    | `removeTag` first tries `findEnclosingMatchingTag`; on miss, tries `findDelimiterInclusiveMatch`. Returns no-op only if both miss.                                                                                                                                                                                                                                          |
| 28  | Styling Engine — Tags           | MD-to-HTML fallback in toggleTag          | After domain conversion, if the tag definition is still a markdown type, `toggleTag` looks up `MARKDOWN_TO_HTML_TAG_MAP` and retries enclosing + delimiter-inclusive detection with the HTML equivalent. Enables toggle-off of previously converted content.                                                                                                                |
| 29  | Styling Engine — Tags           | removeAllTags merge and dedup             | Merges enclosing-tag set and delimiter-inclusive set, deduplicates tag ranges by `openingTagStartOffset` (pass 1), collects unwrap replacements, then deduplicates replacements by `"fromOffset:toOffset"` string key (pass 2), keeping first occurrence (innermost). Also short-circuits when `isFullyInert`.                                                              |
| 30  | Styling Engine — Structure      | Inert zone types                          | Code fences, inline code, math blocks, inline math, table rows (`TABLE_LINE_PATTERN`), horizontal rules (`HORIZONTAL_RULE_PATTERN`). HTML comments are **not** an inert zone.                                                                                                                                                                                               |
| 31  | Styling Engine — Structure      | Inert zone containment                    | A line is only marked inert when it is **fully enclosed** within a zone range (start ≥ zone.start AND end ≤ zone.end). Partial overlap does not trigger inert.                                                                                                                                                                                                              |
| 32  | Styling Engine — Structure      | Unpaired fences                           | `collectFencedBlockRanges` pairs fences sequentially (0+1, 2+3…). | 33  | Styling Engine — Structure      | Atomic ranges (protected)                 | `![[...]]` embeds, `[[wikilinks]]`, `[md links](url)`, `[^footnoteRefs]`, and `#hashtags` (including `/` and `-` chars) are **atomic ranges** — formatting splits around them rather than wrapping inside. Embed pattern wins over wikilink on overlap.                                                                                                                     |
| 34  | Styling Engine — Structure      | Atomic range split                        | When a selection spans an atomic token, the applied format wraps the non-atomic portions on each side separately; the token itself is not modified.                                                                                                                                                                                                                         |
| 35  | Styling Engine — Structure      | Line prefix priority order                | `LINE_PREFIX_PATTERNS` evaluated in fixed order: todo → bullet → numbered → heading → footnoteDefinition → indent. | 36  | Styling Engine — Structure      | Callout prefix two-pass detection         | Strips `> ` first, then checks inner prefix type. `linePrefixType` is set to the inner type (e.g. `todo`); `linePrefix` includes both (`"> - [ ] "`). If only `> ` matches, `linePrefixType` is `callout`.                                                                                                                                                                  |
| 37  | Styling Engine — Structure      | Structure context expands to full lines   | `detectStructureContext` walks backward/forward to the nearest `\n` so the `lines` array always contains complete lines even for mid-line selections.                                                                                                                                                                                                                       |
| 38  | Styling Engine — Structure      | isFullyInert short-circuit                | When all lines are inert, `toggleTag`, `addTag`, and `removeAllTags` all return `{ replacements: [], isNoOp: true }` immediately.                                                                                                                                                                                                                                           |
| 39  | Styling Engine — Multi-line     | Multi-line per-line condition             | Per-line processing is used only when `lines.length > 1` **AND** at least one non-inert line has a structural prefix (heading/list/todo/callout/indent/footnoteDefinition). A plain multi-line selection with no structural prefixes uses full-selection logic.                                                                                                             |
| 40  | Styling Engine — Multi-line     | Per-line toggle all-or-nothing            | If ALL non-inert lines already have the tag → call `toggleTag` (not `removeTag`) on each to remove. If ANY line is missing the tag → call `addTag` only on lines missing it.                                                                                                                                                                                                |
| 41  | Styling Engine — Multi-line     | Per-line content clips to after prefix    | `effectiveStart = Math.max(selectionStart, line.contentStartOffset)`. Formatting is never applied to prefix characters.                                                                                                                                                                                                                                                     |
| 42  | Styling Engine — Multi-line     | Per-line presence checks HTML equivalent  | `lineHasMatchingTag` also checks the HTML equivalent when the tag definition is markdown-domain, matching both `**...**` and `<b>...</b>` forms.                                                                                                                                                                                                                            |
| 43  | Styling Engine — Multi-line     | addTagPerLine unconditional               | `addTagPerLine` calls `addTag` on every line with no per-line presence pre-check. For non-span tags (`<u>`, etc.) this can produce double-wrapping on lines that already have the tag. Only span-with-attributes tags avoid double-wrap (via same-property replacement).                                                                                                    |
| 44  | Styling Engine — Format Painter | copyFormat captures domain                | `CopiedFormat` stores both `tagDefinitions[]` and `domain` (`'markdown'` or `'html'`) at the copy point.                                                                                                                                                                                                                                                                    |
| 45  | Styling Engine — Format Painter | copyFormat reconstructs span tags         | For unknown span enclosures, `tagRangeToTagDefinition` calls `extractStylePropertyFromOpeningTag` then `buildSpanTagDefinition(property, value)`.
